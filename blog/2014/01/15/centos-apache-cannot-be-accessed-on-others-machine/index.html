Centos_apache_cannot_be_accessed_on_others_machine

Jan 15th, 2014

缘起：
在VPS上面安装的CentOS装了个Apache，发现只能本地访问， 其他机器访问不了。

查找原因：
开始猜测是否被墙了， 后来发现墙外也访问不了。 于是猜想是否是被访问墙挡住了。 没有查到log， 尝试停止防火墙，

/etc/init.d/iptables stop

下面是运行log

/etc/init.d/iptables stop

iptables: Setting chains to policy ACCEPT: filter nat [ OK ] iptables: Flushing firewall rules: [ OK ] iptables: Unloading modules: [ OK ]

停止后发现可以访问， 定位成功。

解决方法：

修改iptables配置文件，添加80端口。原先开启了22端口给ssh，复制该行，把22改成80
/etc/sysconfig/iptables

-A INPUT -p tcp -m state –state NEW -m tcp –dport 22 -j ACCEPT

修改后：

-A INPUT -p tcp -m state –state NEW -m tcp –dport 22 -j ACCEPT -A INPUT -p tcp -m state –state NEW -m tcp –dport 80 -j ACCEPT

修改后重启：

/etc/init.d/iptables restart

搞定。 修改过程中， 想过复制网上的解决办法， 诸如： http://os.51cto.com/art/201103/249398.htm 提到的
-A RH-Firewall-1-INPUT -m state –state NEW -m tcp -p tcp –dport 80 -j ACCEPT

修改后无效， 估计是参数次序问题。不如照葫芦画瓢，根据可用的配置来修改。

小结一下iptables配置和命令： 需要root权限 :)

  1. 配置文件 /etc/sysconfig/iptables

  2. 关闭、打开、重启 防火墙

/etc/init.d/iptables stop
/etc/init.d/iptables start
/etc/init.d/iptables restart

参考： http://os.51cto.com/art/201103/249398.htm

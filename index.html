做题-20170409-试出毒酒最少需要多少人

做题-20170409-试出毒酒最少需要多少人

题目：国王举办宴会，48小时后要举行酒会；有酒10000桶但有一桶是毒酒；毒酒喝下去会在之后的第23-24小时内毒死人，即毒发时间[23,24)；国王决定用囚犯来试酒，只要求用最少的囚犯来测试出哪桶是毒酒，问需要最少需要多少囚犯才能保证找出毒酒？ 假设忽略试酒带来的一切损耗；毒死的人不能再试酒了。

题目来源： 万粥帮帮我解一下这道题@joke@smth

思路：

  1. 先做简化版
    这道题的最简化版本就是，10000桶酒有一桶是毒酒，毒酒喝下去就毒发身亡。现在安排囚犯试酒，最少多少人。

这样一个人可以测试一次，区分2桶酒。也就是两桶酒（分别编号0和1），挑第0桶喝，如果毒发这桶有毒，如果没事，那么第1桶酒有毒。

一个人喝一次，可以区分2桶酒。 那么n
个人，就可以区分 2的n次方桶酒。10000桶需要14个人。 即log2(10000)向上取整。

具体如何喝呢？ 要点是：把酒按规律混合起来喝。 我们可以发现这个问题实际上是2进制下一个数需要多少位来表示的问题。每位对应一个人。 把酒编号0到9999，每个酒编号都换成二进制表示，也就是14位数字（0或者1）序列。数字值为1， 那么对应的人，要取该桶酒来喝，当然实际上是把所有需要喝的酒混合起来喝。

  1. 从简化版我们可以总结要点
    a. 按规律混合酒来测试
    b. 假设每个人可以测试m次，就可以区分m+1桶酒。n个人可以区分m+1的n次方桶酒。 问题变成一个m+1进制表示一个数最少需要多少位。

  2. 那么本题条件下， 一个人可以测试多少次 ？ 显然不止一次。 可以发现关键点：毒发时间[23,24)，总时间限制是48小时。

如果第0小时喝，发作时间23-24，如果第1小时喝，发作时间24-25，…。
如果每隔一个小时测试一次，发作时间可以用来识别喝的是哪一种酒。
可以计算到，一个人一共可以测试24次，也就是25进制问题。
之后就可以略过了。

centos: apache server cannot be accessed from other machine

缘起：
在VPS上面安装的CentOS装了个Apache，发现只能本地访问， 其他机器访问不了。

查找原因：
开始猜测是否被墙了， 后来发现墙外也访问不了。 于是猜想是否是被访问墙挡住了。
没有查到log， 尝试停止防火墙，

/etc/init.d/iptables stop

下面是运行log

/etc/init.d/iptables stop

iptables: Setting chains to policy ACCEPT: filter nat [ OK ]
iptables: Flushing firewall rules: [ OK ]
iptables: Unloading modules: [ OK ]

停止后发现可以访问， 定位成功。

解决方法：

修改iptables配置文件，添加80端口。原先开启了22端口给ssh，复制该行，把22改成80
/etc/sysconfig/iptables

-A INPUT -p tcp -m state –state NEW -m tcp –dport 22 -j ACCEPT

修改后：

-A INPUT -p tcp -m state –state NEW -m tcp –dport 22 -j ACCEPT
-A INPUT -p tcp -m state –state NEW -m tcp –dport 80 -j ACCEPT

修改后重启：

/etc/init.d/iptables restart

搞定。 修改过程中， 想过复制网上的解决办法，
诸如： http://os.51cto.com/art/201103/249398.htm 提到的
-A RH-Firewall-1-INPUT -m state –state NEW -m tcp -p tcp –dport 80 -j ACCEPT

修改后无效， 估计是参数次序问题。不如照葫芦画瓢，根据可用的配置来修改。

小结一下iptables配置和命令： 需要root权限 :)

  1. 配置文件 /etc/sysconfig/iptables

  1. 关闭、打开、重启 防火墙

    /etc/init.d/iptables stop
    /etc/init.d/iptables start
    /etc/init.d/iptables restart

参考： http://os.51cto.com/art/201103/249398.htm

问题： 如何配置log？
